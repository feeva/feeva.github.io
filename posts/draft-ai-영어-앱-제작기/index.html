매일 아침 8시, 내 폰으로 메시지가 온다. 오늘 배울 영어 표현이다.

동기: 영어 공부, 꾸준히 하는 게 제일 어렵다

영어는 하면 할수록 느는데, 문제는 꾸준히 하기가 너무 어렵다는 거다. 앱을 깔아보고, 유튜브 채널을 구독해보고, 원서도 사봤지만 결국 흐지부지됐다.

내가 진짜 원하는 건 단순했다. 실제 원어민이 쓰는 관용 표현이나 구동사를 배우는 것. 사전에서는 나오지 않는, 문장을 보고 단어를 조합해도 의미를 유추할 수 없는 그런 표현들. 그리고 그걸 억지로 앉아서 공부하는 게 아니라, 아침에 커피 한 잔 마시면서 자연스럽게 받아보는 것.

그래서 직접 만들기로 했다.

아이디어: AI가 선생님, 안드로이드 폰이 서버

구조는 단순하다:

  1. Hacker News에서 실제 원어민 댓글을 긁어온다
  2. Claude AI가 그 중에서 오늘 배울 만한 표현을 하나 골라준다
  3. Claude가 한국어로 상세한 설명, 예문, 대화 예시를 만들어준다
  4. 매일 정해진 시간에 디스코드로 전송된다
  5. 배운 표현은 DB에 저장되고, 며칠 뒤에 복습 알림이 온다

서버는 따로 빌리지 않았다. 서랍 속에 잠들어 있던 안 쓰는 안드로이드 폰을 꺼냈다. Termux를 설치하고, Python을 올리고, cron으로 스케줄링했다. 전기세 걱정 없이 24시간 돌아가는 개인 서버가 생겼다.

실제 Hacker News 댓글에서 뽑은 표현이라 요즘 IT 업계에서 쓰는 자연스러운 표현들이 많이 나온다.

기술 스택: 외부 패키지 없이 Python 표준 라이브러리만

가장 큰 제약은 안드로이드 ARM 환경이었다. pip로 패키지를 설치하려고 하면 Rust 컴파일러가 없다며 실패하는 경우가 생긴다. 그래서 Python 표준 라이브러리만 쓰기로 했다.

  • urllib — HTTP 요청 (Anthropic API, Discord webhook, HN API 전부)
  • sqlite3 — 학습 기록 저장, 복습 스케줄 관리
  • json — API 응답 파싱
  • argparse — CLI 옵션 (--new, --review, --bulk, --stats)

Anthropic 공식 SDK를 쓰면 편하지만, 안드로이드에서 컴파일이 안 되니 직접 urllib.request로 API를 호출했다. 생각보다 훨씬 단순하다:

def _call_claude(prompt: str, model: str, max_tokens: int) -> dict:
    payload = json.dumps({
        "model": model,
        "max_tokens": max_tokens,
        "messages": [{"role": "user", "content": prompt}],
    }).encode()
    req = urllib.request.Request(
        "https://api.anthropic.com/v1/messages",
        data=payload,
        headers={
            "x-api-key": config.ANTHROPIC_API_KEY,
            "anthropic-version": "2023-06-01",
            "content-type": "application/json",
        },
    )
    with urllib.request.urlopen(req) as res:
        return json.loads(res.read())

AI 역할 분담: Haiku가 고르고, Sonnet이 설명한다

표현 선택과 설명 생성에 서로 다른 모델을 썼다.

표현 선택 (Claude Haiku): 수십 개의 댓글 중에서 오늘 배울 표현 하나를 고르는 작업. 빠르고 저렴하게 처리하면 된다. 단, 프롬프트가 중요하다. "완전히 부적합한(completely unsuitable)"처럼 단어를 직역해도 의미가 통하는 표현은 제외하도록 명시적으로 지시했다. 관용구나 구동사처럼 표면적 의미와 실제 의미가 다른 것만 골라야 학습 효과가 있기 때문이다.

설명 생성 (Claude Sonnet): 선택된 표현을 가지고 상세한 학습 콘텐츠를 만드는 작업. 여기서는 품질이 중요하다. Haiku로 해봤더니 설명이 피상적이라 Sonnet을 쓴다.

망설임 없이 넘어간 결정들

Reddit → Hacker News

처음엔 Reddit 인기 댓글을 쓰려 했다. 다양한 표현이 많이 나오기 때문이다. 그런데 Reddit이 한국에서 403을 뱉기 시작했다. 고민하다 Hacker News Algolia API로 갔다. IT 종사자라면 오히려 HN 표현이 더 실용적일 수도 있다.

Telegram → Discord

처음엔 텔레그램 봇으로 만들었다. 그런데 텔레그램을 잘 안 쓰게 되더라. 디스코드를 평소에 자주 열다 보니 웹훅으로 바꿨다. 봇을 만들 필요도 없고, 웹훅 URL 하나면 끝이다.

앱 설치 비용: 라즈베리파이 제로 vs 안드로이드 폰

라즈베리파이 제로도 고려했다. 그런데 계산해보니 전력 소비는 비슷한데 라즈베리파이는 따로 사야 한다. 안드로이드는 이미 있고, ARM용 Termux 생태계도 생각보다 탄탄하다. 안드로이드가 이겼다.

기능 목록 (현재 완성된 것들)

기능 설명
--new 오늘의 새 표현 전송
--review 간격 반복 복습 (3일 → 7일 → 14일)
--bulk 주간 복습 카드 (최근 7일 표현 한 번에)
--stats 학습 통계 전송
배터리 모니터링 20% 이하 + 미충전 시 디스코드 알림

크론탭 설정 예시:

# 아침 8시: 새 표현
0 8 * * * termux-wake-lock; python ~/daily-language-learn/main.py --new

# 저녁 8시: 복습 (없으면 신규)
0 20 * * * termux-wake-lock; python ~/daily-language-learn/main.py --review

# 일요일 아침 8시: 주간 복습
0 8 * * 0 termux-wake-lock; python ~/daily-language-learn/main.py --bulk

# 15분마다: 배터리 체크
*/15 * * * * ~/daily-language-learn/battery_alert.sh

부딪힌 문제들

안드로이드 Doze 모드

안드로이드는 화면을 끄면 앱 실행을 줄이는 Doze 모드가 있다. Cron이 제시간에 실행되지 않는 문제가 생겼다. 해결책은 영구 웨이크락:

# ~/.termux/boot/start-crond.sh
termux-wake-lock
sshd
crond

부팅 시 웨이크락을 걸어두면 Doze 모드의 영향을 크게 줄일 수 있다. 가끔 몇 분 늦게 오는 건 감수한다.

Discord 2000자 제한

주간 복습 카드에 표현이 여러 개 들어가다 보면 2000자를 넘는다. Discord 웹훅은 2000자 이상이면 400 에러를 뱉는다. 1900자 기준으로 자동 분할해서 여러 메시지로 나눠 보내도록 처리했다.

HN 댓글 수집 속도

처음엔 댓글마다 HTTP 요청을 따로 보냈더니 3분 넘게 걸렸다. Algolia의 items 엔드포인트를 쓰면 게시물 하나의 전체 댓글을 한 번에 가져올 수 있다는 걸 나중에 알았다. 이걸로 바꾸니 8개 게시물 기준 50초 → 8초로 줄었다.

이 글을 읽고 직접 해보고 싶다면

필요한 것:

  • 안 쓰는 안드로이드 폰 (아무거나 괜찮다)
  • Anthropic API 키 — console.anthropic.com 에서 발급. 하루 한두 번 호출 기준으로 월 1~2달러 수준
  • Discord 웹훅 URL — 서버 설정 → 연동 → 웹훅에서 1분이면 만든다
  • Termux — 구글 플레이가 아닌 F-Droid에서 설치해야 crond가 정상 동작한다

코드는 Python 표준 라이브러리만 쓰기 때문에 별도 패키지 설치 없이 바로 돌아간다.

마치며

완성하고 나서 달라진 건 생각보다 단순하다. 아침에 일어나면 디스코드에 메시지가 와 있다. 커피 마시면서 읽는다. 오래 걸리지 않는다.

공부를 위해 앉는 게 아니라, 그냥 읽힌다는 게 제일 큰 차이다.

영어 표현 하나를 외우는 게 목적이 아니라 "아, 이게 이런 뜻이었구나"라는 순간이 쌓이는 것이 목적이라면, 이런 방식이 생각보다 잘 맞는다.
