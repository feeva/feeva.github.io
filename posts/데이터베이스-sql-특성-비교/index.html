데이터베이스 SQL 특성 비교

(2015년 10월 1일 현황: 구글 문서를 원본으로 하여 파일 내보내기를 통해 페이지를 제공하고 있었는데 현재는 파일 형식이 바뀐 등의 문제로 페이지가 안나오고 있다. 시간 나면 다시 수정해야할 듯…)

그 동안 프로젝트를 많이 하다보니 데이터베이스도 여러가지를 사용해봤는데 여러 다른 개발 분야처럼 제품간 차이가 꽤 나는 분야가 바로 이 영역이다. SQL 구문 등에 있어서 크고 작은 여러 차이가 사람을 혼란스럽게 하다보니 2009년에 데이터베이스 SQL 특성 비교 문서를 만든 적이 있다.

벌써 5년 전에 만들다 보니 Oracle 9i, SQL Server 2000 등을 기준으로 했는데 최근에 좀 버전업을 해야겠다는 생각이 들었다. 하지만 계속 바쁜 핑계로 짬이 안나다보니 문서 갱신이 쉽지 않다. 2009년까지는 공부한다는 생각으로도 했던 것 같은데 지금은 일을 더 벌이면 안될 것 같은 상황이다. 그래서 일단 여기 블로그에 공개해서 네티즌 개발자들에게 도움이 되면 좋겠고 좀더 욕심을 내자면 여러 사람의 의견을 수렴해서 좀더 보완하거나 버전업을 하는 것도 좋을 것 같다.

위 웹페이지에도 언급했지만 개인적으로 또는 약간은 회사 동료의 도움을 받아 수작업으로 만든 문서라 100% 완벽하지 않을 수 있다. 또한 SQL의 모든 면모를 비교한 게 아니라 주관적인 판단으로 자주 사용되는 것들만 뽑아놓은 것이라 많은 사람들의 관심 분야와 좀 다를 수도 있음을 감안해주기 바란다.

위 내용에 들어가보면 데이터베이스 제품명마다 댓글 링크가 붙어 있으므로 의견이 있을 때는 가차없이 달아주시면 감사하겠다. 내용 전반에 대한 의견이나 나와 있지 않은 데이터베이스에 대한 의견은 이 웹페이지에 달아주시면 감사하겠다.

그럼 여러 사람에게 조금이나마 도움이 되기를 바라겠다.
